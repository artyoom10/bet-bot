# AGENTS.md

Инструкции для Codex в этом репозитории.

- Работай маленькими шагами: сначала локализуй проблему, затем вноси точечные изменения.
- Сначала анализируй структуру проекта и существующие паттерны, только потом редактируй код.
- Не делай большие переписывания без явной причины или отдельного согласования.
- Сохраняй текущую архитектуру Flask + Supabase REST + Telegram Mini App, если задача прямо не требует другого.
- После значимых изменений обновляй `docs/HANDOFF.md`.
- Если меняется архитектура, поток данных, ключевые модули или внешние интеграции, обновляй `docs/ARCHITECTURE.md`.
- Не печатай секреты, токены, service role keys и реальные значения env-переменных.
- Не редактируй и не трогай `.env`; используй `.env.example` только для описания имён переменных.
- Не откатывай чужие изменения без явной команды пользователя.
- Перед финальным ответом по возможности запускай быстрые проверки: `node --check static/app.js`, `py -m py_compile ...`, `git diff --check`.
